What is a stack?

#1 Stacks include the Last Mile

No more manual or semi-manual steps.

Stacks create a single entry point for comprehensive developer solutions

Easily

  • implement end-to-end developer solutions
  • provide self-service developer solutions
  • increase development velocity

#2 Stacks ensure security

No more unknown sources

Stacks are composed of other stacks (nested workflows)

The components are recursively:

  • visualized through a graph
  • traced from sources to execution
  • validated for security audits

#3 Stacks are publishable

Complex automation workflows with ease.

By layering through stacks, you have flexibility to customize and publish your automations.

Stacks provide:

  • entry points to glue any developer tool
  • unlimited runtime environments
  • parallel executions (concurrency)
